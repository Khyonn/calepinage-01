# Sous-étape 09.1 — Mode `draw` : tracé polygone

## 1. Objectif utilisateur

En mode `draw`, l'utilisateur clique pour placer des sommets successifs. Un aperçu du prochain segment suit le curseur. Avec `Ctrl`, un snap axial simultané en X et Y (seuil 0.75 rem écran (≈ 12 CSS px sur configuration par défaut)) aligne automatiquement sur les sommets existants. À la fermeture du polygone (clic sur le premier sommet ou `Entrée`), l'application demande un nom et enregistre la pièce.

## 4. Comportement attendu

- [ ] Mode `draw` activable. Curseur devient `crosshair`.
- [ ] Clic gauche → place un sommet aux coordonnées monde du curseur.
- [ ] Entre deux vertices, une preview pointillée relie le dernier vertex au curseur.
- [ ] `Ctrl` enfoncé pendant le move → snap axial simultané X et Y sur les vertices existants (seuil 0.75 rem écran (≈ 12 CSS px sur configuration par défaut), conversion via zoom). Lignes guides dessinées pour chaque axe actif (couleur `--accent`).
- [ ] Clic sur le premier vertex (dans un rayon ~`0.75 rem` écran) → ferme le polygone, déclenche la saisie du nom.
- [ ] `Entrée` → même effet que clic premier vertex (au moins 3 sommets requis).
- [ ] `Suppr` → retire le dernier vertex posé.
- [ ] `Échap` → annule entièrement le tracé (tous les vertices sont oubliés).
- [ ] Saisie du nom : Dialog simple avec input + boutons Annuler / Enregistrer. Annuler abandonne les vertices.
- [ ] Enregistrement → dispatch `addRoom({ name, vertices })`, retour au mode `nav`, la nouvelle pièce devient active (`setActiveRoom`).
- [ ] Pendant le tracé, pan canvas (Espace + drag) suspend temporairement l'interaction : pointerup rebranche le tracé.
- [ ] Pièces diagonales autorisées (pas de contrainte 90°).

## 5. Tests

- **BDD** (`src/core/__tests__/snap.feature`) :
  - Scénario : "Snap actif en X uniquement" — cursor proche d'un vertex en X mais éloigné en Y → retourne `snappedX` sur `V.x`, `snappedY = undefined`.
  - Scénario : "Snap simultané X et Y sur deux vertices différents" — cursor proche de `V1.x` et `V2.y` → retourne `snappedX = V1.x`, `snappedY = V2.y`.
  - Scénario : "Seuil en rem écran dépend du zoom" — à `rootFontPx = 16` et zoom 2×, le seuil monde est 6 cm (12/2) ; au zoom 0,5×, 24 cm.
- **BDD** (`tests/features/room-draw.feature`) :
  - Scénario : "Je dessine un rectangle et le nomme" — `selectRooms()` retourne 1 pièce avec le nom et les 4 vertices attendus.
