# Sous-étape 10.3 — Annotations de réutilisation

## 1. Objectif utilisateur

Pour chaque segment de rangée, l'utilisateur voit à gauche la longueur de la chute consommée en première lame (si `xOffset > 0`) et à droite la longueur de la chute générée en fin (si < largeur segment). Un lien visuel (trait ou surbrillance) rapproche les segments source/cible d'une réutilisation.

## 2. Dépendances

- 10.2 (segments rendus).
- Domaine : `computeOffcutLinks(project) → OffcutLink[]` (déjà en place).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/canvas/Row/Segment/Annotation/index.tsx` — texte SVG positionné à côté du segment (gauche ou droite).
- `src/store/selectors.ts` — `selectOffcutLinksForProject()` (memoïsé, si absent).
- `src/components/canvas/Row/OffcutLinkOverlay/index.tsx` — rendu SVG des liens entre segments (trait courbé ou code couleur).

**Modifier :**
- `src/components/canvas/Row/Segment/index.tsx` — monter deux `<Annotation>` conditionnellement :
  - Gauche : si `xOffset > 0`, afficher la longueur consommée.
  - Droite : si la dernière lame du segment a été coupée (chute générée), afficher la longueur restante.

## 4. Comportement attendu

- [ ] Annotation gauche visible uniquement si `xOffset > 0` dans la sous-étape concernée.
- [ ] Annotation droite visible uniquement si la dernière lame est coupée.
- [ ] Texte dans `--font-mono`, taille `--text-xs`, couleur `--text-muted`.
- [ ] Au survol d'une annotation (ou d'une chute), le segment lié (source ou cible) est mis en surbrillance (`--accent-light`).
- [ ] Pendant un drag de segment (jalon 11), les annotations se mettent à jour en temps réel — à ce stade, elles ne sont rendues que sur état statique. **Décision différée** : le comportement exact pendant le drag sera spécifié dans 11.1.
- [ ] Les liens de réutilisation (trait entre segments) sont rendus uniquement sur la pièce active pour ne pas saturer le canvas.
