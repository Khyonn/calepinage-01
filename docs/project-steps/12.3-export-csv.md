# Sous-étape 12.3 — Export CSV

## 1. Objectif utilisateur

Depuis le menu hamburger, l'utilisateur clique "Exporter CSV" et un fichier `.csv` est téléchargé. Il contient : le résumé matière (par type de lame), éventuellement la liste des pièces et rangées, et les liens de réutilisation. Ouvrable directement dans Excel / LibreOffice / Numbers.

## 2. Dépendances

- Jalon 10 ✅ (rangées et segments — pour le résumé).
- 06.2 ✅ (menu hamburger).
- Domaine : `computeSummary`, `computeOffcutLinks` (déjà en place).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/core/exportCsv.ts` — fonction pure `projectToCsv(project: Project) → string`. Produit un CSV multi-sections (cf. format ci-dessous).
- `src/core/__tests__/exportCsv.test.ts` — tests unitaires avec snapshot.
- `src/components/layout/HamburgerMenu/useExportCsv.ts` (ou inline) — hook qui :
  - appelle `projectToCsv(project)`,
  - crée un `Blob`,
  - génère un `URL.createObjectURL` et un `<a download>` cliqué programmatiquement,
  - révoque l'URL dans un `setTimeout` court.

**Modifier :**
- `src/components/layout/HamburgerMenu/index.tsx` — entrée "Exporter CSV" au-dessus de "Supprimer le projet".

## 4. Format CSV proposé

Multi-section, séparateur `;` (compatible Excel FR), encodage UTF-8 avec BOM :

```
# Résumé matière
Type de lame;Quantité;Coût unitaire (€);Coût total (€)
Chêne 1200×190;12;45.00;540.00
...

# Liens de réutilisation
Pièce source;Rangée source;Pièce cible;Rangée cible;Longueur réutilisée (cm)
Salon;2;Salon;4;38
...

# Paramètres de pose
Cale (cm);Écart joints min (cm);Écart rangées min (cm);Première lame min (cm)
1.0;30;20;30
```

Format à trancher / ajuster en cours — prioriser la lisibilité humaine et l'ouverture directe dans un tableur.

## 5. Comportement attendu

- [ ] Clic "Exporter CSV" → téléchargement immédiat d'un fichier nommé `calepinage-<nomProjet>-<dateISO>.csv`.
- [ ] Fichier s'ouvre dans LibreOffice Calc et Excel sans warning d'encodage (BOM UTF-8).
- [ ] Caractères accentués et virgules dans les noms correctement échappés.
- [ ] Si le projet est vide (aucune rangée), l'export reste possible mais le résumé est vide — pas de plantage.
