# Sous-étape 09.2 — Composant `Room` : rendu polygone

## 1. Objectif utilisateur

Chaque pièce enregistrée est rendue sur le canvas sous forme de polygone avec contour et fond semi-transparent. L'apparence reflète l'état : pièce normale, pièce active, pièce atténuée (selon mode).

## 2. Dépendances

- 07.1 (Scene en place).
- 09.1 (pièces peuvent être créées — peut démarrer en parallèle si seed test en place).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/canvas/Room/index.tsx` — rendu d'une pièce : `<polygon points="…" />` avec classes conditionnelles selon l'état.
- `src/components/canvas/Room/Room.module.css` — styles utilisant les tokens `--room-fill`, `--room-stroke`, `--room-active-fill`, `--room-dim-fill`, `--room-dim-stroke`.

**Modifier :**
- `src/components/canvas/Scene/index.tsx` — boucler sur `selectRooms()` et monter un `<Room>` par pièce dans le groupe transformé.
- `src/store/selectors.ts` — ajouter `selectActiveRoomId`, `selectIsRoomActive(id)` (si absents).

## 4. Comportement attendu

- [ ] Une pièce s'affiche dès qu'elle est enregistrée.
- [ ] État normal : fond `--room-fill`, contour `--room-stroke`.
- [ ] État actif (id === `activeRoomId`) : fond `--room-active-fill`, contour accent.
- [ ] État atténué : en mode `edit` sur une autre pièce active, ou en mode `plan` — fond `--room-dim-fill`, contour `--room-dim-stroke`.
- [ ] Curseur `pointer` au survol en mode `nav`.
- [ ] Clic sur la pièce en mode `nav` → la sélectionner et basculer en mode `edit`.
