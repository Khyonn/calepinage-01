# Sous-étape 06.1 — Topbar + layout général

## 1. Objectif utilisateur

Au premier lancement, l'utilisateur voit une interface propre style Excalidraw : topbar en haut, zone canvas centrale vide (placeholder), et un bouton hamburger dans la topbar qui ouvre un menu (vide fonctionnellement à ce stade, câblé en 06.2). Le projet affiché dans la topbar est repris de la session précédente ou créé automatiquement (logique portée par 06.2).

## 2. Dépendances

- Jalon 05 ✅
- Composants atomiques existants : `Button`, `Dialog`, `Drawer` (ne pas recréer).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/layout/Topbar/index.tsx`
- `src/components/layout/Topbar/Topbar.module.css`
- `src/components/layout/AppShell/index.tsx` — conteneur principal (topbar + zone canvas placeholder + drawer).
- `src/components/layout/AppShell/AppShell.module.css`
- `src/components/layout/ModeSelector/index.tsx` — segmented control des 4 modes (nav/draw/edit/plan), tous désactivés à cette étape sauf `nav`.
- `src/components/layout/ModeSelector/ModeSelector.module.css`

**Modifier :**
- `src/App.tsx` — remplacer le `<Demo />` par `<AppShell />`. Déplacer `Demo` sous `src/components/ui/Demo.tsx` si on veut garder la galerie accessible via un flag dev.

**Supprimer :**
- `src/components/ui/Demo.tsx` et son usage — **optionnel**, à n'exécuter que si on ne garde pas la galerie. Sinon, conserver mais la décrocher de la route principale.

## 4. Comportement attendu

- [ ] Topbar hauteur `3rem`, fond `--surface`, ombre `--shadow-sm`.
- [ ] Zone gauche : bouton hamburger (icône, `Button variant="ghost" iconOnly`) + nom du projet (affiche le nom courant — la reprise/auto-create est câblée en 06.2).
- [ ] Zone centre : `ModeSelector` affichant 4 icônes (nav / draw / edit / plan). `edit` est conditionnel (visible seulement si une pièce est active — à ce stade, toujours caché). Seul `nav` est actif et sélectionné par défaut.
- [ ] Zone droite : bouton toggle drawer (icône, ghost, iconOnly). Pas de drawer câblé à ce stade — l'action ne fait rien.
- [ ] Zone canvas placeholder : vide (pas de message ni CTA). 06.2 garantit qu'un projet existe au chargement, donc pas d'état "aucun projet" à gérer ici.
- [ ] Au clic sur le hamburger, un popover vide (ou avec un texte "Menu à venir") s'affiche. Sera câblé en 06.2.
- [ ] Responsive minimal : largeur ≥ `37.5rem` (pas de support mobile spécifique).

Note unités : toutes les tailles CSS hors SVG sont exprimées en `rem`. Les `px` restent réservés aux coordonnées et dimensions à l'intérieur du SVG canvas.
