# Sous-étape 11.2 — Cascade offcut-links au release

## 1. Objectif utilisateur

Quand l'utilisateur relâche un segment dragué, toutes les rangées du même type dans la même pièce dont le `xOffset` dépendait d'une chute de la rangée déplacée voient leur `xOffset` recalculé automatiquement. Le résumé matière (drawer droit, jalon 12) est mis à jour.

## 2. Dépendances

- 11.1 (drag fonctionnel).
- Domaine : `computeOffcutLinks(project)`, `propagateOffcuts(project, movedRowId)` — **à créer si absent**, sinon réutiliser.

## 3. Fichiers à créer / modifier

**Créer (si absents) :**
- `src/core/propagateOffcuts.ts` — fonction pure qui, à partir d'un projet et d'une rangée modifiée, recalcule les `xOffset` des rangées filles (même type de lame, même pièce, postérieures à la rangée source).
- `src/core/__tests__/propagateOffcuts.test.ts`.

**Modifier :**
- `src/store/projectSlice.ts` — dans le reducer `moveSegment`, après application du nouvel `xOffset`, invoquer `propagateOffcuts` et mettre à jour les rangées impactées. Persistance automatique via middleware IDB.
- `src/components/canvas/Row/Segment/useSegmentDrag.ts` — au release, dispatcher une seule action composite plutôt que deux (éviter un re-render intermédiaire).

## 4. Comportement attendu

- [ ] Après release, les rangées du même type postérieures au segment dragué voient leur preview mis à jour si leur `xOffset` était lié à une chute de la rangée source.
- [ ] Les rangées d'un autre type ne sont pas impactées.
- [ ] Les rangées antérieures au segment dragué ne sont pas impactées.
- [ ] Si la propagation rend une rangée fille invalide, la violation s'affiche (10.4) — **aucun blocage**, l'utilisateur voit et corrige.
- [ ] Le résumé matière (drawer droit, implémenté en 12.1) reflète la nouvelle consommation.
- [ ] Un seul commit store : l'état passe de "avant drag" à "après drag + cascade" en un dispatch, pour que l'undo (non implémenté à ce stade, mais anticipé) couvre l'ensemble.

## 5. Tests

- **Unitaires** (`propagateOffcuts.test.ts`) :
  - Scénario : 3 rangées type A + 2 rangées type B, on déplace A[0], seules A[1] et A[2] sont recalculées.
  - Scénario : propagation rendant une rangée fille invalide — l'état reflète la violation mais l'opération réussit.
- **BDD** (`tests/features/offcut-cascade.feature`) :
  - Scénario utilisateur : "Je déplace une rangée et les suivantes du même type s'alignent automatiquement".
- **Manuel** : vérifier visuellement sur une pièce avec 4+ rangées du même type.
