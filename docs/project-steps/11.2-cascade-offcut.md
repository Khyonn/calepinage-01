# Sous-étape 11.2 — Cascade offcut-links au release ✅

## 1. Objectif utilisateur

Quand l'utilisateur relâche un segment dragué, toutes les rangées du même type dans la même pièce dont le `xOffset` dépendait d'une chute de la rangée déplacée voient leur `xOffset` recalculé automatiquement. Le résumé matière (drawer droit, jalon 12) sera mis à jour.

## 2. Dépendances

- 11.1 ✅ (drag fonctionnel).
- Domaine : `computeOffcutLength`, `computeDefaultXOffset` — en place.

## 3. Fichiers créés / modifiés

**Créés :**
- `src/core/propagateOffcuts.ts` — fonction pure (compatible Immer) qui, à partir d'une pièce, des types de lame, des paramètres de pose et de l'ID de la rangée modifiée, recalcule les `segments[0].xOffset` des rangées postérieures du même `plankTypeId` via `computeDefaultXOffset(room, plankType, poseParams, upToRowIndex)`.
- `src/core/__tests__/propagate-offcuts.{feature,steps.ts}` — 3 scénarios BDD (cascade simple, mix de types, source sans chute).
- `src/store/__tests__/segment-drag.{feature,steps.ts}` — 2 scénarios BDD (cascade sur segment 0, pas de cascade sur segment > 0).

**Modifiés :**
- `src/core/addRow.ts` — `computeDefaultXOffset` exporté et étendu avec paramètre optionnel `upToRowIndex` (exclusif, défaut = fin du tableau).
- `src/store/projectSlice.ts` — reducer `updateSegmentOffset` appelle `propagateOffcuts` après mutation du segment, uniquement si `segmentIndex === 0`. Un seul dispatch couvre la mutation + la cascade.

## 4. Comportement implémenté

- [x] Après release, les rangées du même type postérieures au segment dragué voient leurs `xOffset` recalculés si leur consommation de chute dépendait de la rangée source.
- [x] Les rangées d'un autre type ne sont pas impactées.
- [x] Les rangées antérieures à la rangée source ne sont pas impactées.
- [x] Si la propagation rend une rangée fille invalide, la violation s'affiche (10.4) — aucun blocage.
- [x] Cascade déclenchée **uniquement pour `segmentIndex === 0`** — les segments intérieurs d'une pièce concave ne produisent pas la chute « fin de rangée ».
- [x] Un seul dispatch `updateSegmentOffset` : la mutation et la cascade sont appliquées dans le même draft Immer, un seul re-render.

## 5. Tests

- **BDD domaine** (`propagate-offcuts.feature`) : cascade 3 rangées même type, cascade filtrée par `plankTypeId`, source sans chute → downstream à 0.
- **BDD store** (`segment-drag.feature`) : dispatch sur segment 0 cascade, dispatch sur segment > 0 ne cascade pas.
- **Manuel** : vérifié sur une pièce 400×300 avec 4+ rangées du même type.
