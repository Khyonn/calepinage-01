# Sous-étape 06.4 — Dark mode

## 1. Objectif utilisateur

L'utilisateur peut choisir entre les thèmes light, dark et system via le menu hamburger. Le choix est persisté et restauré au prochain chargement. Tous les écrans respectent le thème actif.

## 2. Dépendances

- 06.1 (topbar + hamburger), 06.2 (menu hamburger peuplé).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/persistence/theme.ts` — adapter `readTheme` / `writeTheme` (clé `calepinage.theme` en `localStorage`), `resolveTheme` (résolution `system` → `light`/`dark`), `applyTheme` (pose l'attribut `data-theme` sur `<html>`).
- `src/hooks/useTheme.ts` — hook partagé exposant `{ theme, setTheme }`. En mode `system`, enregistre un listener `matchMedia('(prefers-color-scheme: dark)')` et réapplique `data-theme` quand la préférence OS change.
- `src/components/layout/ThemeSwitcher/index.tsx` — bloc de menu avec `RadioGroup` (Clair / Sombre / Système). Utilise `useTheme`.

**Modifier :**
- `src/components/layout/HamburgerMenu/index.tsx` — ajouter un séparateur puis `<ThemeSwitcher />` inline en fin de menu (pas de sous-menu flyout : 3 options tiennent dans le menu principal, on évite les bugs d'outside-click imbriqué).
- `src/main.tsx` — au boot, lire `readTheme()` et appliquer `applyTheme(resolveTheme(...))` **avant** `createAppStore()` / `createRoot` pour éviter un flash.
- `src/index.css` — confirmer que les tokens `--bg`, `--surface`, `--text`, etc. ont bien un overload `[data-theme="dark"]` (déjà en place depuis étape 04 — vérifier).

**Choix de persistance :** `localStorage` (clé `calepinage.theme`) plutôt qu'un object store `meta` IndexedDB. Motifs : accès synchrone (permet d'appliquer le thème avant le premier render sans attendre l'ouverture d'IDB), scalaire trivial (pas besoin de Structured Clone), cohérent avec `lastProjectId`. IDB reste réservé aux données projet.

## 4. Comportement attendu

- [ ] Au premier lancement, thème = `system`. L'attribut `data-theme` reflète `prefers-color-scheme` (light ou dark).
- [ ] Choix `light` → attribut `data-theme="light"`, persisté.
- [ ] Choix `dark` → attribut `data-theme="dark"`, persisté.
- [ ] Choix `system` → écoute `matchMedia('(prefers-color-scheme: dark)')` et met à jour l'attribut dynamiquement.
- [ ] Au reload, le thème choisi est restauré avant le premier render (pas de flash de thème opposé).
- [ ] Tous les panneaux, dialogs, et le placeholder canvas respectent le thème actif (vérification visuelle).
