# Sous-étape 14.1 — Primitive `Tooltip` (design system)

## 1. Objectif utilisateur

L'utilisateur survole (ou met le focus clavier sur) un bouton iconOnly de la topbar ou un badge de violation sur le canvas, et voit s'afficher un tooltip stylé, lisible en clair comme en sombre, qui explique ce que fait l'élément ou ce que la violation signifie. Plus de texte SVG `<title>` brut sans style.

## 2. Dépendances

- Jalon 06 ✅ (topbar, boutons iconOnly avec `aria-label`).
- Sous-étape 10.4 ✅ (`ViolationBadge` en place, premier consommateur).
- Aucune dépendance externe (pas de Floating UI, pas de Radix).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/ui/Tooltip/index.tsx` — composant `<Tooltip content="…" placement?="top|bottom|left|right" delay?={300}>{children}</Tooltip>`. Wrap un trigger unique. Portal dans `document.body`. Affiche un overlay HTML positionné via `getBoundingClientRect()` du trigger. ARIA `role="tooltip"` + génération d'un id liée au trigger via `aria-describedby`.
- `src/components/ui/Tooltip/Tooltip.module.css` — fond et texte issus des tokens (`--surface-inverse`, `--text-on-inverse`), `font-size: 0.8125rem`, `border-radius: var(--radius-sm)`, `padding: var(--space-2) var(--space-3)`, `box-shadow: var(--shadow-md)`, `transition: opacity 120ms ease-out`. Flèche optionnelle via `::after`.
- `src/components/ui/Tooltip/useTooltip.ts` — hook gérant `open`, `triggerRef`, position calculée (clamp dans `window.innerWidth/Height`), timer d'ouverture (cleanup au unmount), listeners `pointerenter`/`pointerleave`/`focusin`/`focusout`/`keydown` Escape.
- `src/components/ui/Tooltip/__tests__/tooltip.feature` + `tooltip.steps.ts` — scénarios BDD : show on hover après délai, hide on leave, show on focus immédiat, Escape ferme, ARIA correct.

**Modifier :**
- `src/components/canvas/Row/Segment/ViolationBadge/index.tsx` — supprimer `<title>{title}</title>`. Wrapper le `<g>` racine du badge avec un overlay HTML positionné en absolu sur la viewport. Décision de design : utiliser un overlay HTML *en dehors du SVG* synchronisé via la position écran du badge (lue par `getBoundingClientRect()` au hover), pas un `<foreignObject>` (cassé sur Firefox + difficultés de positionnement zoom-aware). Le composant `Tooltip` doit fournir une variante adaptée à un trigger SVG (passe `triggerRef` à `<g>`).
- `src/components/layout/Topbar/index.tsx` — wrapper chaque `<Button iconOnly aria-label="…">` avec `<Tooltip content="…">`.
- `src/components/layout/HamburgerMenu/index.tsx` — pas de modification dans cette sous-étape (le menu déroulant a déjà son texte, le bouton hamburger est ciblé via topbar).
- `src/index.css` — si manquants, ajouter les tokens `--surface-inverse`, `--text-on-inverse`, `--shadow-md` (ils existent probablement déjà via le design system jalons 04/06 ; vérifier avant d'en créer).
- `docs/features/ui-specifications.md` — section "Tooltip" : props, placement, délai, accessibilité.

**Ne pas modifier :**
- `Annotation` SVG des planches (annotations chiffrées, pas de tooltip prévu en 14.1).
- Drawer / SummaryDrawer.

## 4. Comportement attendu

- [ ] Au survol d'un trigger pendant ≥ 300ms, le tooltip apparaît.
- [ ] À la sortie du pointeur, le tooltip disparaît immédiatement.
- [ ] Au focus clavier (Tab), le tooltip apparaît **sans délai**.
- [ ] À la perte de focus (Tab vers un autre élément), il disparaît.
- [ ] Escape ferme le tooltip ouvert (et empêche sa réouverture tant que le pointeur n'a pas quitté le trigger).
- [ ] Le tooltip est positionné selon `placement` ; si dépasse la viewport, clamp pour rester visible (pas de flip auto en 14.1, juste clamp).
- [ ] Le trigger porte `aria-describedby={id}` pendant l'ouverture, retiré à la fermeture.
- [ ] Le tooltip porte `role="tooltip"` et l'`id` correspondant.
- [ ] Multiples tooltips ne s'empilent pas — un seul ouvert à la fois (un trigger qui ouvre ferme les autres).
- [ ] Sur badge de violation : le contenu du tooltip est le message multi-lignes existant (`messageForViolation` joint par `\n`, rendu avec `white-space: pre-line`).
- [ ] Sur boutons topbar : contenu = `aria-label` actuel.
- [ ] Pas de tooltip si `content` est vide ou `null`.
- [ ] Contraste WCAG AA respecté en dark mode (vérification visuelle).

## 8. Risques & arbitrages

- **Overlay HTML hors SVG** vs `foreignObject` : choix HTML overlay. Position calculée via `getBoundingClientRect()` du trigger (DOM ou SVG `<g>` — `getBoundingClientRect` fonctionne sur les deux). Plus simple, plus stylable, compatible cross-browser.
- **Pas de placement auto-flip** : si la viewport est trop petite, on clamp. Si l'expérience est insuffisante en pratique, ouvrir une 14.1.bis ultérieure.
- **Délai 300ms hard-codé par défaut** : prop `delay?` permet override ponctuel. Pas de config globale.
- **Mobile / tactile** : hover non disponible. La sous-étape ne livre PAS de tap-to-show. À traiter dans un jalon dédié si demande utilisateur.
- **Test infra** : si l'ajout de `@testing-library/react` est jugé trop lourd pour cette sous-étape, fallback sur tests unitaires du hook `useTooltip` uniquement, et test manuel pour le rendu. Décision à arbitrer en début de session.
