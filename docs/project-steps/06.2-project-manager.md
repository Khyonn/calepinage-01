# Sous-étape 06.2 — Gestionnaire de projets (reprise + auto-create)

## 1. Objectif utilisateur

L'utilisateur ouvre l'app et est immédiatement sur un projet utilisable : soit le dernier ouvert, soit un projet vide créé à la volée si aucun n'existe. Depuis le menu hamburger, il peut créer un nouveau projet, ouvrir un projet existant, ou supprimer le projet courant. Le nom du projet est éditable inline dans la topbar.

## 2. Dépendances

- 06.1 (topbar + hamburger rendus).
- Actions store déjà disponibles : `projectActions.create`, `projectActions.open`, `projectActions.delete`, selectors associés.

## 4. Comportement attendu

- [ ] Au chargement, un projet est **toujours** courant (reprise ou auto-create). Pas d'état "aucun projet".
- [ ] Hamburger → popover avec : Nouveau projet, Ouvrir un projet, Supprimer le projet.
- [ ] "Supprimer le projet" : si c'est le dernier projet, la suppression est suivie d'un auto-create immédiat (logique symétrique au boot).
- [ ] "Nouveau projet" → dialog avec input nom (validation : non vide, trim). Créer → dispatch `createProjectThunk`, ferme dialog, devient courant.
- [ ] "Ouvrir un projet" → dialog listant les projets (depuis `projectListSlice`, donc sans charger chacun). Clic → `openProjectThunk(id)` charge le projet complet.
- [ ] Nom projet éditable inline dans la topbar (composant dédié). Commit au blur ou Entrée, annulation sur Échap.
- [ ] `lastProjectId` est mis à jour dans localStorage à chaque ouverture/création/suppression.
- [ ] Deux onglets : dernier à écrire `lastProjectId` gagne — pas de synchro cross-tab (documenter).
