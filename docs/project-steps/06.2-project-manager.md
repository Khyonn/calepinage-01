# Sous-étape 06.2 — Gestionnaire de projets (reprise + auto-create)

## 1. Objectif utilisateur

L'utilisateur ouvre l'app et est immédiatement sur un projet utilisable : soit le dernier ouvert, soit un projet vide créé à la volée si aucun n'existe. Depuis le menu hamburger, il peut créer un nouveau projet, ouvrir un projet existant, ou supprimer le projet courant. Le nom du projet est éditable inline dans la topbar.

## 2. Dépendances

- 06.1 (topbar + hamburger rendus).
- Actions store déjà disponibles : `projectActions.create`, `projectActions.open`, `projectActions.delete`, selectors associés.
