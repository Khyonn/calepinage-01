# Sous-étape 06.3 — Panneau contextuel en mode `nav` (paramètres de pose + catalogue)

## 1. Objectif utilisateur

Dans le panneau contextuel haut-gauche en mode `nav`, l'utilisateur règle les paramètres de pose du projet (4 champs) et gère le catalogue de types de lames (ajouter, éditer, supprimer). Les suppressions de types utilisés sont bloquées.

## 2. Dépendances

- 06.1 (shell), 06.2 (un projet doit exister pour voir le panneau).
- Actions store : `projectActions.updatePoseParams`, `addPlankType`, `updatePlankType`, `deletePlankType`. Selectors : `selectPoseParams`, `selectCatalog`, `selectPlankTypeUsage` (à créer si absent).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/layout/ContextualPanel/index.tsx` — conteneur collapsible (haut-gauche) commutant son contenu selon `state.ui.mode`. À ce stade n'a qu'un seul rendu : mode `nav`.
- `src/components/layout/ContextualPanel/ContextualPanel.module.css`
- `src/components/panels/PoseParamsForm/index.tsx` — 4 champs (`Slider` ou `Input` number) : cale, sawWidth, minPlankLength, minRowGap. Dispatch `updatePoseParams` au blur / changement debounced.
- `src/components/panels/PlankCatalog/index.tsx` — liste des types, bouton "+ Ajouter un type", formulaire d'ajout inline ou dialog, boutons édition / suppression par ligne.
- `src/components/panels/PlankCatalog/PlankTypeRow/index.tsx` — une ligne de la liste.
- `src/components/panels/PlankCatalog/PlankTypeForm/index.tsx` — formulaire ajout / édition (name, length, width, pricing avec RadioGroup unité/lot + champs conditionnels, description).

**Modifier :**
- `src/store/selectors.ts` — ajouter `selectPlankTypeUsage(plankTypeId)` qui retourne le nombre de rangées utilisant ce type.
- `src/components/layout/AppShell/index.tsx` — monter `<ContextualPanel />` si un projet est courant.

## 4. Comportement attendu

**Panneau en général :**
- [ ] Collapsible (clic sur l'en-tête → collapse/expand, persisté en session via `ui` slice).
- [ ] Visible uniquement si un projet est courant ET si mode = `nav` (à ce stade, toujours si projet courant).
- [ ] Ne pousse pas le canvas — flotte par-dessus (position absolue, `box-shadow: var(--shadow-lg)`).

**Paramètres de pose :**
- [ ] 4 champs avec valeurs par défaut : cale 0,5 cm, sawWidth 0,1 cm, minPlankLength 30 cm, minRowGap 15 cm.
- [ ] Saisie quantifiée à 0,1 cm (min, max, step).
- [ ] Dispatch sur blur ou après debounce 300 ms.
- [ ] Changement recalcule automatiquement les résultats (via selectors déjà en place).

**Catalogue :**
- [ ] Liste affichant pour chaque type : nom, dimensions (`length × width`), pricing résumé, badge d'utilisation ("Utilisé dans N rangées" ou "Non utilisé").
- [ ] Boutons action par ligne : Éditer (ouvre le formulaire), Supprimer (disabled si utilisé, confirmation sinon).
- [ ] Formulaire d'ajout / édition :
  - name (required, non vide)
  - length (required, > minPlankLength)
  - width (required, > 0)
  - pricing : RadioGroup unité (prix par lame) / lot (prix + taille lot) — champs conditionnels
  - description (optionnel)
  - Si édition et type utilisé : dimensions (length, width) passent en lecture seule. Les autres champs restent modifiables.
- [ ] Bouton "Ajouter un type" toujours disponible.

## 5. Tests

- **BDD** (`tests/features/nav-panel.feature`) :
  - Scénario : "Je modifie la cale de dilatation" — le selector `selectPoseParams().cale` est mis à jour.
  - Scénario : "J'ajoute un type de lame au catalogue" — `selectCatalog()` contient le nouveau type.
  - Scénario : "Je tente de supprimer un type utilisé dans une rangée" — la suppression est refusée, `selectCatalog()` inchangé.
  - Scénario : "J'édite un type utilisé : les dimensions sont verrouillées" — après dispatch `updatePlankType` avec length/width différents, `selectCatalog()` montre les anciennes dimensions (si la règle est appliquée côté store) OU le formulaire empêche la soumission (si appliquée côté UI ; à choisir et documenter).
