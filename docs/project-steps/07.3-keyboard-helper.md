# Sous-étape 07.3 — Raccourcis clavier + helper contrôles

## 1. Objectif utilisateur

L'utilisateur consulte à tout moment un panneau d'aide en bas-gauche listant les raccourcis disponibles dans le mode actif. Les raccourcis globaux (hors pan/zoom souris déjà couverts par 07.2) sont gérés via un hook partagé avec guard focus formulaire.

**Pas de pan au clavier** : la navigation canvas se fait à la souris (07.2). Le clavier sert uniquement aux raccourcis d'action (modes, annulation, validation…) qui seront ajoutés au fil des jalons.

## 2. Dépendances

- 07.2 (événements canvas en place).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/hooks/useKeyboardShortcuts.ts` — hook partagé qui enregistre des raccourcis avec guard focus formulaire (`input`, `textarea`, `select`, `[contenteditable]`). API : `useKeyboardShortcuts([{ key, ctrl?, shift?, handler }])`.
- `src/components/layout/HelperPanel/index.tsx` — panneau bas-gauche collapsible. Contenu initial : raccourcis souris disponibles à ce stade (pan souris, zoom Ctrl+molette, scroll horizontal Shift+molette, Espace+drag). Enrichi au fil des jalons par les modes (draw/edit/plan ajoutent leurs raccourcis quand ils deviennent actifs).
- `src/components/layout/HelperPanel/HelperPanel.module.css`

**Modifier :**
- `src/components/layout/AppShell/index.tsx` — monter `<HelperPanel />` en bas-gauche.

## 4. Comportement attendu

- [ ] Helper panel visible par défaut, collapsible via clic sur son en-tête (état persisté en session — volatile, pas IDB).
- [ ] Libellés accessibles en français : "Déplacer la vue" (pan), "Zoomer" (Ctrl + molette), etc. Pas de jargon anglais ("Pan", "Scroll horizontal").
- [ ] Le contenu du helper dépend du mode actif via `selectMode()`. À ce stade, seul `nav` a du contenu spécifique.
- [ ] Les raccourcis enregistrés via `useKeyboardShortcuts` sont automatiquement inactifs si le focus est sur un élément de formulaire.
- [ ] API du hook extensible — aucune dépendance directe à une action métier à ce stade, le helper ne déclenche rien de custom en 07.3.

## 5. Tests

- **Unit** : test du hook `useKeyboardShortcuts` (simuler focus sur un `<input>`, vérifier que le handler n'est pas appelé).
- Protocole manuel : ouvrir le helper, le replier, vérifier qu'il reste cohérent au changement de mode.
