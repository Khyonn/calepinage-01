# Sous-étape 12.2 — Tableau des liens de réutilisation ✅

## 1. Objectif utilisateur

Dans le drawer droit, sous le tableau résumé, l'utilisateur voit la liste des liens de réutilisation : rangée source (chute générée) → rangée cible (chute consommée), longueur réutilisée. Au survol d'une ligne, les deux rangées concernées sont mises en surbrillance sur le canvas.

## 2. Dépendances

- 12.1 ✅ (drawer en place).
- 10.3 ✅ (rendu des annotations de réutilisation).
- Domaine : `computeOffcutLinks(rooms, catalog, poseParams) → OffcutLink[]` (déjà en place).

**Modifiés :**

## 4. Choix techniques

- **ID d'un lien** : dérivé à la volée `${sourceRowId}→${targetRowId}` via `offcutLinkId(link)`. Le domaine garantit l'unicité (une seule chute source par rangée) → pas de collision.
- **`uiSlice.hoveredOffcutLinkId: string | null`** : éphémère, non persisté (cohérent avec le middleware IDB qui ignore les actions UI).
- **Hover** : `onPointerEnter` / `onPointerLeave` sur `<tr>` → dispatch Redux. Le selector `selectHighlightedRowIds` est mémoïsé via `createSelector`.

## 5. Comportement

- [x] Tableau listant les `OffcutLink` du projet entier.

- [x] Tableau vide si aucun lien : message "Pas encore de chutes réutilisées".
