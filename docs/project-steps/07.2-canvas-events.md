# Sous-étape 07.2 — Événements natifs pan / zoom

## 1. Objectif utilisateur

L'utilisateur navigue dans le canvas à la souris :
- Molette → scroll vertical
- Shift + molette → scroll horizontal
- Ctrl + molette → zoom vers le curseur
- Espace + clic gauche drag **OU** clic milieu drag → pan

## 2. Dépendances

- 07.1 (Scene + viewport + grille en place).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/canvas/Scene/useCanvasEvents.ts` — attache des listeners **natifs** (`addEventListener` sur le SVG) :
  - `wheel` avec `{ passive: false }` pour pouvoir `preventDefault()` sur Ctrl+wheel.
  - `pointerdown`, `pointermove`, `pointerup`, `pointercancel` pour le pan.
  - `keydown` / `keyup` global pour traquer l'état `Espace` enfoncé.
- Le hook retourne éventuellement des classes CSS dynamiques (cursor `grab`/`grabbing`).

**Modifier :**
- `src/components/canvas/Scene/index.tsx` — appeler `useCanvasEvents()` et binder les ref(s) nécessaires.
- `src/store/uiSlice.ts` — si des reducers manquent (`zoomAt`, `pan`), les ajouter.

## 4. Comportement attendu

- [ ] Molette → pan vertical en unités écran (scroll), conversion via viewport.
- [ ] Shift + molette → pan horizontal.
- [ ] Ctrl + molette → zoom autour du curseur. Facteur `1.1` par cran. Bornes `min 0.1`, `max 10` (à confirmer par test).
- [ ] Canvas **focusé** (tabindex=0 + focus) et Espace enfoncé → curseur immédiatement en `grab` (indication visuelle qu'un drag clic gauche déclencherait un pan), même sans mouvement souris.
- [ ] Espace (maintenu) + clic gauche drag → pan. Curseur `grabbing` pendant le drag.
- [ ] Clic molette drag → pan (même logique, sans Espace).
- [ ] Relâchement Espace / bouton → fin du pan, curseur redevient normal.
- [ ] Si le focus est sur un `input`, `textarea`, `select`, `[contenteditable]`, Espace ne déclenche pas le changement de curseur ni le pan.
- [ ] Le canvas doit être focusable (`tabindex="0"` sur le SVG racine) pour que le changement de curseur au keydown Espace fonctionne sans passer d'abord par un clic.
- [ ] Aucun `React.SyntheticEvent` utilisé pour ces interactions — tout passe par `addEventListener` natif (confirmer via relecture).

## 5. Tests

- BDD/unit : limité (les events DOM natifs sont pénibles à tester sans jsdom étendu). Documenter un protocole manuel.
- **Protocole manuel** :
  1. Ouvrir l'app, tester chaque raccourci (molette, shift+molette, ctrl+molette, espace+drag, clic milieu).
  2. Vérifier que le zoom est bien centré sur le curseur (pas sur l'origine).
  3. Vérifier que Ctrl+wheel sur un input ne déclenche pas le zoom.
