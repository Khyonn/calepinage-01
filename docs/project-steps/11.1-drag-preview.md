# Sous-étape 11.1 — Drag segment + preview temps réel ✅

## 1. Objectif utilisateur

En mode `edit` sur une pièce active, l'utilisateur appuie sur le bouton gauche au-dessus d'un segment, glisse horizontalement, et voit les lames de ce segment se repositionner en temps réel. Le curseur passe en `grabbing`. Au relâchement, la nouvelle valeur de `xOffset` est persistée.

## 2. Dépendances

- 10.2 ✅ (segments rendus et cliquables).
- 10.4 ✅ (visualisation des violations — nécessaire pour le feedback pendant drag).
- Domaine : `fillRow`, `validateRow` — déjà en place.

## 3. Fichiers créés / modifiés

**Créés :**
- `src/components/canvas/Row/useSegmentDrag.ts` — hook local : `previewOffsets`, `draggingIndex`, handlers pointer (pointerdown/move/up/cancel + Escape). Dispatch `updateSegmentOffset` au release.

**Modifiés :**
- `src/components/canvas/Row/index.tsx` — intègre le hook, recalcule `fillRow` localement pour le segment dragué, recalcule les violations live (via `validateRow`) pendant le drag, passe un snapshot des planks committées à `Segment` pour figer les annotations.
- `src/components/canvas/Row/Segment/index.tsx` — accepte `onPointerDown`, `dragActive`, `dragging`, `snapshotPlanks`. Utilise `snapshotPlanks` pour les annotations si fourni, les `planks` live pour les rects et les violations.
- `src/components/canvas/Row/Segment/Segment.module.css` — classes `.interactive` (curseur `grab`, `pointer-events: auto`) et `.dragging` (curseur `grabbing`, lames opacité 0.7 + contour `--accent`).

## 4. Comportement implémenté

- [x] Hover sur un segment en mode `edit` + pièce active : curseur `grab`.
- [x] PointerDown bouton gauche : démarrer drag, curseur `grabbing`, opacité segment 70 %, contour `--accent`.
- [x] PointerMove : `xOffset = initialOffset − (deltaClientX / zoom)`, quantifié à 0,1 cm, borné à `[0, plankType.length − 0,1]`.
- [x] Preview : `fillRow(previewOffset, ...)` recalculé localement sans dispatch.
- [x] **Politique non-bloquante** (décision validée) : le preview suit librement le curseur ; les violations sont recalculées live (`validateRow` sur les planks preview) et rendues en rouge + badge. Au release, la valeur courante est commitée telle quelle.
- [x] PointerUp : dispatch `updateSegmentOffset({ xOffset })`, retour à l'opacité normale, curseur `grab`.
- [x] Échap pendant drag : annule (aucun dispatch), état local réinitialisé.
- [x] `pointercancel` système : même comportement qu'Échap.
- [x] **Annotations figées** (décision validée) : textes de mesure lus depuis `snapshotPlanks` (état committé) pendant toute la durée du drag, reprennent les valeurs live au release.
