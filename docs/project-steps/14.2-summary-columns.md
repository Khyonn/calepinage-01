# Sous-étape 14.2 — SummaryTable : colonnes Lames / Quantité / Coût unit. / Coût total

## 1. Objectif utilisateur

Le tableau **Par type de lame** du drawer résumé devient plus lisible : chaque ligne fusionne le type et le nombre de lames dans la première colonne, exprime la quantité dans **l'unité d'achat réelle** (unités ou lots), et annote le coût unitaire avec la base de prix (`/ unité` ou `/ lot`). L'utilisateur identifie en un coup d'œil ce qu'il doit commander chez son fournisseur.

## 8. Risques & arbitrages

- **Densité visuelle** : la cellule col 1 passe de 2 lignes à 3 lignes (count + nom + dims). Vérifier que ça ne casse pas l'alignement des autres colonnes — `vertical-align: top` sur les `td` num.
- **Format lot multi-ligne** : `2 lots de 6 (12 unités)` peut tenir sur une ligne ; en cas de drawer étroit, autoriser le wrap ou passer à deux lignes (line 1: `2 lots de 6`, line 2 muted: `12 unités`). Choix retenu : deux lignes (cohérent avec col 1).
- **Locale fr-FR** : utiliser `Intl.NumberFormat('fr-FR', { style: 'currency', currency: 'EUR' })` ou conserver `toFixed(2).replace('.', ',')` simple. Le formatter actuel utilise `toFixed(2)` avec point — à harmoniser sur virgule pour cohérence française.
- **Pas de tooltip** sur les colonnes — réservé à 14.3 si nécessaire.
