# Sous-étape 08.1 — Mode `plan` + import image

## 1. Objectif utilisateur

L'utilisateur active le mode `plan` dans la topbar, importe une image (via file picker ou drag & drop sur le canvas), règle son opacité (slider 0–100 %) et sa rotation (4 boutons 0° / 90° / 180° / 270°). L'image s'affiche en arrière-plan du canvas sous la grille.

## 2. Dépendances

- Jalon 07 ✅ (Scene + viewport).
- Store : `projectActions.setBackgroundPlan` (créer si absent — doit accepter `{ imageFile, opacity, rotation, x, y }`).

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/canvas/BackgroundPlan/index.tsx` — composant SVG `<image>` avec les bonnes props (href = object URL de `imageFile`, opacity, rotation).
- `src/components/canvas/BackgroundPlan/BackgroundPlan.module.css` (si styling nécessaire).
- `src/components/panels/PlanImportForm/index.tsx` — contenu du panneau contextuel en mode `plan` : file picker, drag & drop zone, slider opacité, boutons rotation, bouton "Supprimer le plan".
- `src/components/panels/PlanImportForm/useImagePicker.ts` — logique de lecture du fichier (File API), validation (type image, taille max raisonnable).

**Modifier :**
- `src/components/layout/ContextualPanel/index.tsx` — ajouter le rendu du `PlanImportForm` quand `mode === 'plan'`.
- `src/components/layout/ModeSelector/index.tsx` — activer le bouton `plan` (précédemment désactivé).
- `src/store/projectSlice.ts` — reducer `setBackgroundPlan`, `updateBackgroundPlanOpacity`, `updateBackgroundPlanRotation`, `deleteBackgroundPlan`.
- `src/components/canvas/Scene/index.tsx` — monter `<BackgroundPlan />` comme premier enfant du `<g transform>` si `selectBackgroundPlan()` retourne une valeur.
- `src/core/types.ts` — rien (BackgroundPlan existe déjà).
- `src/persistence/db.ts` — vérifier que `saveProject` persiste bien le `File` via l'object store `files` (déjà en place).

## 4. Comportement attendu

- [ ] Mode `plan` activable depuis la topbar. Panneau contextuel affiche le PlanImportForm.
- [ ] Drag & drop d'une image sur le canvas → import (même que file picker).
- [ ] Types acceptés : `image/png`, `image/jpeg`, `image/webp`. Rejeter les autres avec message d'erreur.
- [ ] Taille max suggérée : 10 Mo (à confirmer). Au-delà, avertir mais ne pas bloquer.
- [ ] Slider opacité 0 % → 100 %, défaut 60 %.
- [ ] 4 boutons rotation en RadioGroup style segmented (0° sélectionné par défaut).
- [ ] Image affichée sous la grille (`<image>` avant le `<rect fill="url(#grid)" />` ? non : grille est dans le fond plein canvas, l'image doit être **dans** le groupe transformé pour suivre pan/zoom — la grille reste visible par-dessus grâce à une grille semi-transparente OU en la déplaçant dans le groupe monde). **Décision à trancher dans la sous-étape**. La spec demande : grille toujours visible, image en dessous des pièces.
- [ ] Bouton "Supprimer le plan" → dispatch `deleteBackgroundPlan`.
- [ ] Les pièces (quand elles existeront, en 09) sont atténuées `opacity: 0.3` en mode `plan`.
