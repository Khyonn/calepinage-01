# Sous-étape 08.2 — Repositionnement du plan (drag x/y)

## 1. Objectif utilisateur

En mode `plan`, l'utilisateur clique-glisse l'image pour la repositionner sur le canvas. La position (x, y) est persistée. Les pan/zoom du canvas restent fonctionnels en parallèle.

## 2. Dépendances

- 08.1 (plan importable et rendu).
- Store : reducer `updateBackgroundPlanPosition(x, y)` à ajouter.

## 3. Fichiers à créer / modifier

**Créer :**
- `src/components/canvas/BackgroundPlan/useBackgroundPlanDrag.ts` — gère le drag en mode `plan` : `pointerdown` sur l'image → capture position de départ, `pointermove` → met à jour x/y via dispatch, `pointerup` → release.

**Modifier :**
- `src/components/canvas/BackgroundPlan/index.tsx` — câbler le hook quand `mode === 'plan'`.
- `src/store/projectSlice.ts` — reducer `updateBackgroundPlanPosition`.
- `src/components/canvas/Scene/useCanvasEvents.ts` — prioriser le drag image sur le pan de canvas en mode `plan` (si pointer commence sur l'image).

## 4. Comportement attendu

- [ ] En mode `plan`, le curseur sur l'image devient `grab`, `grabbing` pendant le drag.
- [ ] Drag → translation de l'image. Coordonnées stockées en cm monde.
- [ ] Pan canvas (Espace + drag, clic milieu) reste disponible même en mode `plan`, mais ne déclenche PAS le drag image.
- [ ] Après un reload, l'image est repositionnée au x/y persisté.
- [ ] Hors mode `plan`, l'image n'est pas interactive (ni curseur `grab`, ni drag possible).

## 5. Tests

- **BDD** : étendre `background-plan.feature` avec un scénario "Je repositionne le plan" — après dispatch `updateBackgroundPlanPosition`, `selectBackgroundPlan().x` et `.y` sont mis à jour.
