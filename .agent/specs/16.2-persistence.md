# Spec 16.2 — Foundations persistence

## Objectif

Refonte structurelle de la couche persistence + serialize JSON, sans changement fonctionnel visible pour l'utilisateur. Cette session pose les bases pour les sessions suivantes (routing, mode unifié, undo).

Trois changements groupés :

1. **Schéma IDB allégé** : split en deux tables au lieu de six. `projects` (un blob assemblé par projet) + `files` (`File` plan de fond). On stocke côté Redux le `fileId` plutôt que l'objet `File` (sérialisable).
2. **JSON sans IDs** : suppression des UUID dans le format d'export. Le catalogue est référencé par index dans les rangées. L'image est embarquée en base64 dans `backgroundPlan`. Drop `exportedAt`. Garder `version`.
3. **Fix des leaks de couche** :
   - Déplacer les types `*Record` de `src/store/types.ts` vers `src/persistence/`.
   - Sortir l'accès à `localStorage`/`document` de `persistence/lastProjectId.ts` et `persistence/theme.ts` : ces deux fichiers déménagent dans `src/hooks/` ou sont lus par un thunk Redux.

## Pré-requis

- Branche : `rewrite/v2`.
- Lire avant : `.agent/conventions.md`, `.agent/patterns/new-store-test.md`.
- Cette session **casse la rétro-compatibilité IDB locale**. Pendant la branche `rewrite/v2`, vider IDB manuellement (DevTools → Application → IndexedDB → Clear). Migration `v3 → v4` à écrire pour le merge final vers `main`.

## Détails par changement

### 1. Schéma IDB v4

Nouveau schéma :

```ts
interface CalepinageDB extends DBSchema {

}
```

`ProjectRecord` (dans `src/persistence/types.ts`) :

```ts
export interface ProjectRecord {
  id: string
  name: string
  poseParams: PoseParams
  catalog: PlankTypeRecord[]
  rooms: RoomRecord[]
  backgroundPlan?: BackgroundPlanRecord
}

export interface RoomRecord {
  id: string
  name: string
  vertices: Point[]
  yOffset: number
```

### 2. Migration v3 → v4

Dans `db.ts`, bloc `if (oldVersion < 4)` :

```ts
if (oldVersion < 4) {
  // Lire toutes les données dans l'ancien schéma
  const oldProjects = await tx.objectStore('projects').getAll()
  const oldRooms = await tx.objectStore('rooms').getAll()
  const oldRows = await tx.objectStore('rows').getAll()
  const oldPlankTypes = await tx.objectStore('plankTypes').getAll()
  const oldPlans = await tx.objectStore('plans').getAll()

  // Reconstruire un ProjectRecord agrégé pour chaque projet
  for (const oldProject of oldProjects) {
    const projectRooms = oldRooms.filter(r => r.projectId === oldProject.id).map((room) => ({
      id: room.id,
      name: room.name,
      vertices: room.vertices,
      yOffset: room.yOffset ?? 0,
      rows: oldRows
        .filter(r => r.roomId === room.id)
        .sort((a, b) => (a.order ?? 0) - (b.order ?? 0))
        .map(row => ({ id: row.id, plankTypeId: row.plankTypeId, segments: row.segments })),
    }))
    const projectCatalog = oldPlankTypes
      .filter(pt => pt.projectId === oldProject.id)
      .map(pt => ({ id: pt.id, name: pt.name, length: pt.length, width: pt.width, pricing: pt.pricing, description: pt.description }))
    const oldPlan = oldPlans.find(p => p.projectId === oldProject.id)
    const backgroundPlan = oldPlan ? {
      id: oldPlan.id, fileId: oldPlan.fileId,
      calibration: oldPlan.calibration, opacity: oldPlan.opacity, rotation: oldPlan.rotation,
      x: oldPlan.x, y: oldPlan.y,
    } : undefined

    const newRecord: ProjectRecord = {
      id: oldProject.id,
      name: oldProject.name,
      poseParams: oldProject.poseParams,
      catalog: projectCatalog,
      rooms: projectRooms,
      backgroundPlan,
    }
    // Le store `projects` change de schéma → on doit le recréer.
    // Stratégie : suppression des anciens stores, création des nouveaux.
  }
  // Créer les nouveaux stores après lecture
  // (idb permet de manipuler les stores dans l'upgrade callback)
  db.deleteObjectStore('projects')
  db.deleteObjectStore('rooms')
  db.deleteObjectStore('rows')
  db.deleteObjectStore('plankTypes')
  db.deleteObjectStore('plans')
  db.createObjectStore('projects', { keyPath: 'id' })
  // `files` existe déjà depuis v1, on le garde tel quel
  // Réinjecter les nouveaux records
  const projectsStore = tx.objectStore('projects')
  for (const newRecord of newRecords /* construits ci-dessus */) {
    await projectsStore.put(newRecord)
  }
}
```

**Attention** : `db.deleteObjectStore` doit être appelé sur l'instance `db` (callback `upgrade`), pas via `tx`. Vérifier la doc `idb` pour la séquence exacte. Tester localement la migration en chargeant un projet v3 réel avant de pousser.

### 3. JSON v2 — format cible

Avant (v1) :
```json
{
  "version": 1,
  "exportedAt": "2026-04-28T12:34:56Z",
  "project": {
    "id": "uuid",
    "rooms": [{ "id": "uuid", "rows": [{ "plankTypeId": "uuid", ... }] }],
    "catalog": [{ "id": "uuid", "name": "Chêne", ... }]
  },
  "image": { "name": "...", "mimeType": "...", "dataUrl": "data:..." }
}
```

Après (v2) :
```json
{
  "version": 2,
  "project": {
    "name": "Mon projet",
    "poseParams": { ... },
    "catalog": [
      { "name": "Chêne", "length": 120, "width": 12, "pricing": {...}, "description": "" }
    ],
    "rooms": [
      {
        "name": "Salon",
        "vertices": [...],
        "yOffset": 0,
        "rows": [
          { "plankTypeIndex": 0, "segments": [{ "xOffset": 0 }] }
        ]
      }
    ],
    "backgroundPlan": {
      "calibration": {...}, "opacity": 1, "rotation": 0, "x": 0, "y": 0,
      "image": { "name": "plan.png", "mimeType": "image/png", "dataUrl": "data:image/png;base64,..." }
    }
  }
}
```

Différences :
- Drop tous les IDs (`project.id`, `room.id`, `row.id`, `plankType.id`, `backgroundPlan.id`).
- Drop `exportedAt`.
- `row.plankTypeId` → `row.plankTypeIndex` (index dans `catalog`).
- `image` est embarqué dans `backgroundPlan.image` (au lieu d'être au niveau racine).
- Bump `version: 2`.

À l'import :

```ts
function projectFromJson(json: SerializedProjectV2): Project {
  const projectId = crypto.randomUUID()
  // Régénérer les IDs des plankTypes
  const catalog: PlankType[] = json.project.catalog.map(pt => ({
    ...pt, id: crypto.randomUUID(), projectId,
  }))
  // Reconstruire les rangées avec mapping index → id
  const rooms: Room[] = json.project.rooms.map(room => ({
    ...room, id: crypto.randomUUID(), projectId,
    yOffset: clampYOffset(room.yOffset ?? 0, catalog),
    rows: room.rows.map(row => ({
      ...row, id: crypto.randomUUID(), roomId: '', /* à remplir */
      plankTypeId: catalog[row.plankTypeIndex]?.id ?? catalog[0]?.id ?? '',
    })),
  }))
  // Recoller roomId dans rows
  for (const room of rooms) for (const row of room.rows) row.roomId = room.id

  // Image base64 → File
  let backgroundPlan: BackgroundPlan | undefined
  if (json.project.backgroundPlan) {
    const { image, ...rest } = json.project.backgroundPlan
    const file = image ? dataUrlToFile(image.dataUrl, image.name, image.mimeType) : undefined
    backgroundPlan = {
      ...rest, id: crypto.randomUUID(), projectId,
      ...(file ? { imageFile: file } : {}),
    }
  }

  return { id: projectId, name: json.project.name, poseParams: json.project.poseParams, catalog, rooms, backgroundPlan }
}
```

Helper `dataUrlToFile(dataUrl, name, mimeType): File` à écrire dans `core/utils/file.ts` (créer si absent).

### 6. Migration `lastProjectId` et `theme` vers `hooks/`

Avant :
```ts
// persistence/lastProjectId.ts
export const localStorageLastProjectId = { read, write, clear }
```

Après :
```ts
// hooks/useLastProjectId.ts
import { useCallback, useSyncExternalStore } from 'react'

const KEY = 'calepinage.lastProjectId'

const subscribe = (callback: () => void) => {
  window.addEventListener('storage', callback)
  return () => window.removeEventListener('storage', callback)
}

export function useLastProjectId() {
  const value = useSyncExternalStore(
    subscribe,
    () => { try { return localStorage.getItem(KEY) } catch { return null } },
    () => null,
  )
  const set = useCallback((id: string) => { try { localStorage.setItem(KEY, id) } catch {} }, [])
  const clear = useCallback(() => { try { localStorage.removeItem(KEY) } catch {} }, [])
  return { value, set, clear }
}
```

Idem pour `theme.ts` → `useTheme.ts` :
```ts
// hooks/useTheme.ts
import { useEffect, useState } from 'react'

type Theme = 'light' | 'dark' | 'system'
const KEY = 'calepinage.theme'

function readTheme(): Theme {
  try { return (localStorage.getItem(KEY) as Theme) ?? 'system' } catch { return 'system' }
}

function resolveTheme(theme: Theme): 'light' | 'dark' {
  if (theme !== 'system') return theme
  return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'
}

export function useTheme() {
  const [theme, setTheme] = useState<Theme>(() => readTheme())
  useEffect(() => {
    document.documentElement.setAttribute('data-theme', resolveTheme(theme))
    try { localStorage.setItem(KEY, theme) } catch {}
  }, [theme])
  return { theme, setTheme }
}
```

`App.tsx` appelle `useTheme()` au mount → effet de bord (set attribute) appliqué.

`main.tsx` n'appelle plus `applyTheme()`.

- **Risque `useSyncExternalStore`** : valider que React 19 le supporte natif (oui).
