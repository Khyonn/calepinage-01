# Spec 16.3 — Fondations canvas + URL

## Objectif

Session S3 : poser les fondations canvas/URL pour la suite de la refonte. Cinq chantiers indépendants livrés ensemble :

1. **Routing maison** History API (~30 LOC) — module statique `router = { go(url), subscribe(listener) }`. Pas de hook `useRouterSync`. La synchronisation URL → store est faite par un middleware Redux qui s'abonne au router au boot.
2. **Topbar** réduite : à gauche menu hamburger + nom projet éditable, au centre sélecteur mode, à droite **bouton flottant** d'ouverture du drawer résumé. L'état d'ouverture du drawer est **local au composant**, pas dans Redux.
3. **Theme grid** canvas (clair/sombre) via custom properties + `localStorage` direct, hook `useTheme`.
4. **Auto-fit** zoom/pan, déclenché **au changement de projet uniquement** (pas au changement de pièce). Pas de `ResizeObserver`. Signature pure `computeAutoFit(project, { width, height })`.
5. **Migration drag** vers `@use-gesture/vanilla` : Scene (pan/zoom/pinch) + `useBackgroundPlanDrag` (pattern preview/commit, dispatch unique au release).

## Pré-requis

- Branche : `rewrite/v2`. Spec 16.2 mergée (couches `persistence/` propres, `lastProjectId` en module statique + middleware, `theme` en hook React).
- Lire avant : `.agent/conventions.md`, `.agent/patterns/new-component.md`, `.agent/patterns/new-store-test.md`, `.agent/patterns/new-bdd-test.md`.
- Décisions arbitrées (ne pas re-discuter) :
  - Pas de lib routing (`wouter`, `react-router`). Module maison `src/router/index.ts` ~30 LOC.
  - **API router minimale** : `{ go(url), subscribe(listener) }`. Pas de hook React de synchronisation, pas de `useEffect` URL↔state. Les composants appellent `router.go('/...')` directement et lisent l'état projet/mode/pièce via `useAppSelector(selectXxx)`.
  - L'**ordonnance de redirection** est centralisée dans une fonction pure `resolveRoute(url, list, lastProjectId) → RouteState | RedirectTo`. Le middleware applique soit le state final, soit déclenche un `router.go(redirect)` en mode `replace` puis re-évalue.
  - L'**état d'ouverture du drawer** (résumé droite) reste local au composant qui le possède (`useState`). Pas de `uiActions.toggleDrawer`. Pas de champ `drawerOpen` dans `uiSlice`.
  - `@use-gesture/vanilla` (pas `/react`).
  - Pattern « preview pendant drag, dispatch unique au release ».
  - `localStorage` direct pour le thème (pas Redux, pas IDB).
  - `redux-undo` arrive en S6 → ne pas l'introduire ici.
  - Modes URL whitelistés à S3 : `nav` | `edit` | `plan` | `draw`. À S4 (spec 16.4) la whitelist devient `main` | `plan` | `draw`. La conversion `nav`/`edit` → `main` est gérée par 16.4, pas ici.
  - **Auto-fit déclenché uniquement au changement de projet courant** (`project.current.id`). Pas au changement de pièce. Pas de `ResizeObserver` — la mise à l'échelle reste statique tant que l'utilisateur ne change pas de projet ou ne pan/zoom à la main.

## Détails par changement

### A. Routing — module statique + middleware

#### A.1 Types et shape

`src/router/types.ts` :

```ts
// À S3 (avant 16.4) : whitelist URL = nav | edit | plan | draw.
// À S4 (après 16.4) : whitelist URL = main | plan | draw.
// La spec 16.4 ajustera ce type. Pour S3, on conserve les 4 modes URL existants.
export type RouteMode = 'nav' | 'edit' | 'plan' | 'draw'

export interface RouteState {
  projectId: string
  mode: RouteMode
  roomId: string | null
}
```

Convention URL :
- `/` — racine, redirection automatique (voir A.3).
- `/:projectId/:mode` — projet + mode.
- `/:projectId/:mode/:roomId` — projet + mode + pièce active.

Pas d'URL `/new` ni `DEFAULT_ROUTE` exposé : la racine `/` est toujours redirigée vers `/:projectId/:mode` (création projet si liste vide, sinon dernier projet).

#### A.2 `router` — module statique

`src/router/index.ts` :

```ts
type Listener = (url: string) => void

const listeners = new Set<Listener>()
let popstateBound = false

function emit() {
  const url = window.location.pathname
  for (const l of listeners) l(url)
}

function ensurePopstate() {
  if (popstateBound) return
  window.addEventListener('popstate', emit)
  popstateBound = true
}

export const router = {
  go(url: string, opts?: { replace?: boolean }) {
    if (window.location.pathname === url) return
    if (opts?.replace) window.history.replaceState(null, '', url)
    else window.history.pushState(null, '', url)
    emit()
  },
  subscribe(listener: Listener): () => void {
    ensurePopstate()
    listeners.add(listener)
    listener(window.location.pathname) // emit current url to new subscriber
    return () => listeners.delete(listener)
  },
}
```

API minimale, pas de hook React. Tout consommateur (composant ou middleware) appelle `router.go(url)` directement.

#### A.3 `resolveRoute` — fonction pure (toutes les redirections)

`src/router/resolveRoute.ts` :

```ts
import type { ProjectsListEntry } from '@/store/types'
import type { RouteMode, RouteState } from './types'

const MODES: RouteMode[] = ['nav', 'edit', 'plan', 'draw']
const DEFAULT_MODE: RouteMode = 'nav'

export type RouteResolution =
  | { kind: 'apply'; state: RouteState }
  | { kind: 'redirect'; url: string }

export function resolveRoute(
  pathname: string,
  list: ProjectsListEntry[],
  lastProjectId: string | null,
): RouteResolution {
  const segs = pathname.split('/').filter(Boolean)

  // Cas 1 : racine "/" → redirige vers dernier projet, sinon création
  if (segs.length === 0) {
    const target = lastProjectId && list.some(p => p.id === lastProjectId) ? lastProjectId : null
    if (target) return { kind: 'redirect', url: `/${target}/${DEFAULT_MODE}` }
    // Aucun projet existant → middleware créera un projet et redirigera ensuite
    return { kind: 'redirect', url: '/__create__' } // sentinel, voir middleware
  }

  const [projectId, modeRaw, roomId] = segs

  // Cas 2 : projet inconnu → racine
  if (!list.some(p => p.id === projectId)) {
    return { kind: 'redirect', url: '/' }
  }

  // Cas 3 : pas de mode dans l'URL → ajoute le mode par défaut
  if (!modeRaw) {
    return { kind: 'redirect', url: `/${projectId}/${DEFAULT_MODE}` }
  }

  // Cas 4 : mode invalide → coerce vers 'nav'
  const mode: RouteMode = MODES.includes(modeRaw as RouteMode)
    ? (modeRaw as RouteMode)
    : DEFAULT_MODE
  if (mode !== modeRaw) {
    const tail = roomId ? `/${mode}/${roomId}` : `/${mode}`
    return { kind: 'redirect', url: `/${projectId}${tail}` }
  }

  // Cas 5 : roomId présent — la validité de l'index est testée par le middleware
  // (qui a accès au state Redux). Ici on remonte tel quel ; si la pièce n'existe
  // pas, le middleware redirigera vers /:projectId/:mode (voir A.5).
  return {
    kind: 'apply',
    state: { projectId, mode, roomId: roomId ?? null },
  }
}
```

Règles couvertes :
- `/` → `/${lastProjectId}/${DEFAULT_MODE}` ou sentinel `'/__create__'` (à charge du middleware d'orchestrer la création).
- Projet inconnu → racine.
- Mode absent → ajout du mode par défaut.
- Mode invalide (ex. `/abc/foo`) → coerce en `nav`.
- `roomId` invalide : géré par le middleware (qui a le state, fait l'arbitrage `/:projectId/:mode`).

Toutes les redirections passent en `replace: true` (pas d'historique pollué).

#### A.4 `buildUrl`

`src/router/buildUrl.ts` :

```ts
import type { RouteState } from './types'

export function buildUrl(s: RouteState): string {
  const tail = s.roomId ? `/${s.mode}/${s.roomId}` : `/${s.mode}`
  return `/${s.projectId}${tail}`
}
```

Pas de cas `null projectId` — `RouteState.projectId` est toujours non-nul (l'utilisateur ne « voit » jamais l'état pré-redirect).

#### A.5 `routerMiddleware` — synchronisation URL ↔ store

`src/store/middleware/routerMiddleware.ts` :

```ts
import type { Middleware } from '@reduxjs/toolkit'
import { router } from '@/router'
import { resolveRoute } from '@/router/resolveRoute'
import { buildUrl } from '@/router/buildUrl'
import { lastProjectId } from '@/persistence/lastProjectId'
import { selectActiveRoomId, selectCurrentProjectId, selectInteractionMode, selectProjectList, selectRooms } from '@/store/selectors'
import { uiActions } from '@/store/uiSlice'
import { projectActions } from '@/store/projectSlice'
import { openProjectThunk, createProjectThunk } from '@/store/thunks'

export const routerMiddleware: Middleware = (api) => {
  let lastEmittedUrl: string | null = null
  let unsubscribe: (() => void) | null = null

  const apply = (url: string) => {
    if (url === lastEmittedUrl) return // evite la boucle
    const state = api.getState()
    const list = selectProjectList(state)
    const r = resolveRoute(url, list, lastProjectId.read())

    if (r.kind === 'redirect') {
      // Cas spécial : sentinel création
      if (r.url === '/__create__') {
        const id = api.dispatch(createProjectThunk({ name: 'Nouveau projet' }))
        // openProjectThunk déclenchera setActiveProject → loop store → URL plus bas
        return
      }
      router.go(r.url, { replace: true })
      return
    }

    const { projectId, mode, roomId } = r.state
    if (selectCurrentProjectId(state) !== projectId) {
      void api.dispatch(openProjectThunk(projectId))
    }
    api.dispatch(uiActions.setMode(mode))

    // Garde-fou : roomId absent du store → strip
    const rooms = selectRooms(api.getState())
    if (roomId && !rooms.some(rm => rm.id === roomId)) {
      router.go(`/${projectId}/${mode}`, { replace: true })
      return
    }
    api.dispatch(uiActions.setActiveRoom(roomId))
  }

  // Lazy subscribe au premier appel : à l'init du store le router est prêt.
  unsubscribe = router.subscribe(apply)

  return (next) => (action) => {
    const result = next(action)
    // store → URL : actions qui changent projet/mode/pièce
    const watched = ['project/setActiveProject', 'ui/setMode', 'ui/setActiveRoom']
    if (typeof action === 'object' && action !== null && 'type' in action && watched.includes(action.type as string)) {
      const state = api.getState()
      const projectId = selectCurrentProjectId(state)
      const mode = selectInteractionMode(state)
      const roomId = selectActiveRoomId(state)
      if (projectId) {
        const url = buildUrl({ projectId, mode: mode as never, roomId })
        if (url !== window.location.pathname) {
          lastEmittedUrl = url
          router.go(url)
        }
      }
    }
    return result
  }
}
```

Point clé : un seul middleware fait `URL → store` (via `router.subscribe`) et `store → URL` (via `next(action)` puis comparaison). Pas de hook React. Pas de `useEffect`. Le composant ne sait rien du router au-delà d'un `router.go(url)` ponctuel s'il a besoin de naviguer programmatiquement.

#### A.6 Lecture côté composant

Aucun composant n'appelle `router.subscribe`. Pour connaître mode/projet/pièce :

```ts
const mode = useAppSelector(selectInteractionMode)
const projectId = useAppSelector(selectCurrentProjectId)
const roomId = useAppSelector(selectActiveRoomId)
```

Pour naviguer :

```ts
import { router } from '@/router'
router.go(`/${projectId}/draw`)
// ou plus explicite via une action store qui déclenche le middleware :
dispatch(uiActions.setMode('draw')) // → middleware push l'URL
```

**Préférence** : passer par les actions Redux quand un état doit changer. Le `router.go()` direct est réservé aux cas qui ne sont pas représentés par une action (ex. lien interne, cas exceptionnel).

#### A.7 Bootstrap

`src/store/index.ts` :

```ts
export async function createAppStore() {
  const list = await listProjects()
  const initialId = lastProjectId.read()
  const preloaded = initialId ? await loadProject(initialId) : null

  const store = configureStore({
    reducer: rootReducer,
    preloadedState: { project: { list, current: preloaded } },
    middleware: (g) => g().concat(idbMiddleware, lastProjectIdMiddleware, routerMiddleware),
```

### D. Auto-fit canvas

#### D.1 Logique pure `core/autoFit.ts`

Signature **deux paramètres** : un `Project` (la fonction extrait elle-même les vertices pertinents) + un `viewport: { width, height }`. Pas de `marginRatio` paramétrable — constante interne (10 %).

```ts
import type { Project, Viewport } from '@/core/types'

const MARGIN_RATIO = 0.1

export function computeAutoFit(project: Project, viewport: { width: number; height: number }): Viewport {
  const allVertices = project.rooms.flatMap(r => r.vertices)
  if (allVertices.length === 0 || viewport.width === 0 || viewport.height === 0) {
    return { zoom: 1, panX: viewport.width / 2, panY: viewport.height / 2 }
  }
  const xs = allVertices.map(v => v.x), ys = allVertices.map(v => v.y)
  const minX = Math.min(...xs), maxX = Math.max(...xs)
  const minY = Math.min(...ys), maxY = Math.max(...ys)
  const w = maxX - minX || 1, h = maxY - minY || 1
  const usableW = viewport.width * (1 - 2 * MARGIN_RATIO)
  const usableH = viewport.height * (1 - 2 * MARGIN_RATIO)
  const zoom = Math.min(usableW / w, usableH / h)
  const cx = (minX + maxX) / 2, cy = (minY + maxY) / 2
  return {
    zoom,
    panX: viewport.width / 2 - cx * zoom,
    panY: viewport.height / 2 - cy * zoom,
  }
}
```

> **Note `d3-scale`** : non obligatoire. La logique linéaire est triviale (3 lignes). Pas d'ajout de dépendance pour ça. Si une session future a besoin de scales non-linéaires (log, time), `d3-scale` sera reconsidéré.

Tests `core/__tests__/auto-fit.feature` (logique pure) :

- Projet avec deux pièces décalées, viewport 1000×1000 → bounding box global utilisé pour le fit.
- Projet sans pièce → viewport identité centré.
- Viewport 0×0 → identité.

#### D.2 Hook `useAutoFit`

Trigger **uniquement** sur changement de `project.current.id`. Pas au changement de pièce active. Pas de `ResizeObserver`.

```ts
import { useEffect, useRef } from 'react'
import { useAppSelector } from '@/hooks/redux'
import { selectCurrentProject, selectCurrentProjectId } from '@/store/selectors'
import type { ViewportApi } from '@/hooks/useViewport'
import { computeAutoFit } from '@/core/autoFit'

interface Args {
  api: ViewportApi
  // Le Scene fournit la taille connue du viewport (lue une fois au mount via getBoundingClientRect)
  // ou bien lue au moment du fit. Pour S3 on lit au moment du fit via un ref svg.
  svgRef: React.RefObject<SVGSVGElement | null>
}

export function useAutoFit({ api, svgRef }: Args): void {
  const projectId = useAppSelector(selectCurrentProjectId)
  const project = useAppSelector(selectCurrentProject)
  const setViewportRef = useRef(api.setViewport)
  setViewportRef.current = api.setViewport

  useEffect(() => {
    if (!project) return
    const svg = svgRef.current
    if (!svg) return
    const r = svg.getBoundingClientRect()
    setViewportRef.current(computeAutoFit(project, { width: r.width, height: r.height }))
  }, [projectId])
}
```

**Pas** de `ResizeObserver`, **pas** de `userPannedRef` à coordonner avec `useCanvasEvents`. Si l'utilisateur redimensionne sa fenêtre, le fit n'est pas re-déclenché — il garde son zoom/pan courant. C'est volontaire : éviter de casser une vue construite à la main.

Si l'utilisateur veut re-fitter manuellement : à prévoir un bouton « recadrer » dans une session ultérieure (hors S3).

#### E.2 `useCanvasEvents` réécrit

```ts
import { useEffect, useRef, useState, type RefObject } from 'react'
import { Gesture } from '@use-gesture/vanilla'
import type { ViewportApi } from '@/hooks/useViewport'
import { useSpaceKey } from '@/hooks/useSpaceKey'

export type CursorMode = '' | 'spaceDown' | 'panning'

export function useCanvasEvents(
  svgRef: RefObject<SVGSVGElement | null>,
  api: ViewportApi,
  userPannedRef: RefObject<boolean>,
): { cursorMode: CursorMode } {
  const [isPanning, setIsPanning] = useState(false)
  const { spaceDownRef, spaceDown } = useSpaceKey()
  const apiRef = useRef(api); apiRef.current = api

  useEffect(() => {
    const svg = svgRef.current
    if (!svg) return
    const gesture = new Gesture(svg, {
      onWheel: ({ event, delta: [dx, dy], ctrlKey, shiftKey }) => {
        event.preventDefault?.()
        if (ctrlKey) {
          const factor = dy < 0 ? 1.1 : 1 / 1.1
          apiRef.current.zoomAt(
            (event as WheelEvent).offsetX, (event as WheelEvent).offsetY, factor,
          )
        } else if (shiftKey) {
          apiRef.current.pan(-dy, 0)
        } else {
          apiRef.current.pan(-dx, -dy)
        }
      },
      onDrag: ({ first, last, delta: [dx, dy], buttons }) => {
        const middle = (buttons & 4) === 4
        const allow = spaceDownRef.current || middle
        if (!allow) return
        if (first) setIsPanning(true)
        apiRef.current.pan(dx, dy)
        userPannedRef.current = true
        if (last) setIsPanning(false)
      },
      onPinch: ({ origin: [ox, oy], offset: [scale], memo }) => {
        const prev = (memo as number | undefined) ?? scale
        const factor = scale / prev
        const rect = svg.getBoundingClientRect()
        apiRef.current.zoomAt(ox - rect.left, oy - rect.top, factor)
        return scale
      },
    }, {
      drag: { pointer: { buttons: [1, 4] } },
      eventOptions: { passive: false },
    })
    return () => gesture.destroy()
  }, [svgRef, spaceDownRef, userPannedRef])

  const cursorMode: CursorMode = isPanning ? 'panning' : spaceDown ? 'spaceDown' : ''
  return { cursorMode }
}
```

Notes :
- Vérifier la version installée pour la signature des handlers (`delta` vs `movement`).
- Le pan reste local React (`setViewport`), pas de dispatch Redux. OK avec la règle preview/commit.
- Conserver fallback molette milieu (`buttons & 4`) en parallèle de espace + clic gauche.
- `event.preventDefault()` requiert `passive: false`.

### F. `useBackgroundPlanDrag` — preview / commit

#### F.1 Cible

```ts
import { useEffect, useRef, useState, type RefObject } from 'react'
import { Gesture } from '@use-gesture/vanilla'
import { useAppDispatch } from '@/hooks/redux'
import { projectActions } from '@/store/projectSlice'
import { useSpaceKey } from '@/hooks/useSpaceKey'
import { useViewportContext } from '@/components/canvas/Scene/ViewportContext'
import type { BackgroundPlan } from '@/core/types'
import type { InteractionMode } from '@/store/types'

interface Params {
  imageRef: RefObject<SVGImageElement | null>
  plan: BackgroundPlan | null
  mode: InteractionMode
}

export function useBackgroundPlanDrag({ imageRef, plan, mode }: Params): { draft: { x: number; y: number } | null } {
  const dispatch = useAppDispatch()
  const { spaceDownRef } = useSpaceKey()
  const { viewport } = useViewportContext()
  const zoomRef = useRef(viewport.zoom); zoomRef.current = viewport.zoom
  const [draft, setDraft] = useState<{ x: number; y: number } | null>(null)
  const startRef = useRef<{ x: number; y: number } | null>(null)

  useEffect(() => {
    const img = imageRef.current
    if (!img || !plan || mode !== 'plan') return
    const gesture = new Gesture(img, {
      onDrag: ({ first, last, movement: [mx, my], cancel, buttons, event }) => {
        if (spaceDownRef.current || (buttons & 1) !== 1) { cancel(); return }
        event.stopPropagation()
        if (first) startRef.current = { x: plan.x, y: plan.y }
        const start = startRef.current
        if (!start) return
        const z = zoomRef.current || 1
        const next = { x: start.x + mx / z, y: start.y + my / z }
        if (last) {
          dispatch(projectActions.updatePlan(next))
          setDraft(null)
          startRef.current = null
        } else {
          setDraft(next)
        }
      },
    }, { eventOptions: { passive: false } })
    return () => gesture.destroy()
  }, [imageRef, plan, mode, dispatch, spaceDownRef])

  return { draft }
}
```

Garanties :
- **Aucun dispatch** Redux pendant le drag.
- **Un seul** `dispatch(updatePlan)` au release.
- Le composant consommateur lit `draft` et l'applique en `transform` SVG si présent.

#### F.2 Adaptation `BackgroundPlan/index.tsx`

```tsx

const y = draft?.y ?? plan.y
return <image ref={imageRef} x={x} y={y} ... />
```
