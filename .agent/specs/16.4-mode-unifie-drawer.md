# Spec 16.4 — Mode unifié `main` + drawer accordéon

## Contexte & objectif

Session S4 du jalon 16. Trois changements fonctionnellement liés, regroupés parce qu'ils touchent les mêmes points : la sémantique des modes, le panneau latéral, et l'affichage des rangées.

1. **Mode unifié `main`** : fusionner `nav` + `edit`. La distinction « regarder » vs « éditer une pièce » disparaît côté UI : un clic sur une pièce en mode `main` la sélectionne *et* ouvre son panneau d'édition. Modes finaux : `main`, `plan`, `draw`.
2. **Drawer accordéon** : le panneau contextuel droit devient un drawer accordéon en mode `main` : sections collapsibles (Pièce, Rangées, Catalogue, Résumé), une seule ouverte à la fois, état purement local (`useState`).
3. **Cleanup `RoomEditPanel`** : retirer la mention « ajustez les sommets via les croix sur le canvas » + raccourcis Ctrl/Shift. Conserver nom de pièce, yOffset, liste des rangées, ajout de rangée, suppression de la pièce.
4. **Inversion ordre rangées** : vérification — l'ordre actuel est déjà cohérent (rangée 0 en haut du panneau ET en haut du canvas SVG). Cette sous-section est une checklist de vérification.

## Détails par changement

### A. Mode unifié `main` (fusion `nav` + `edit`)

**État initial** : `'main'`. Comportement clic pièce :

- `mode === 'main'` + clic sur pièce → `setActiveRoom(room.id)`. Pas de `setMode`.
- `mode === 'main'` + clic sur le fond → `setActiveRoom(null)` (déselectionner).
- Bouton « Pièces » dans `ModeSelector` : si déjà en `main`, dispatch `setActiveRoom(null)`. Sinon dispatch `setMode('main')`.

**Label** : `'Vue'` → `'Pièces'`. Icône `Hand` conservée.

**`RoomSwitcher`** : `selected = mode === 'edit'` → `selected = activeRoomId !== null`.

**Pièges** :
- Vérifier que le routing S3 émet `'main'` dans l'URL et plus jamais `'nav'`/`'edit'`.
- Vérifier que `useRouterSync` accepte `'main'` comme mode valide (mapper bidirectionnel à supprimer puisque le store est désormais `'main'`).

### B. Drawer accordéon

**Composant `<Accordion>` réutilisable** dans `src/components/ui/Accordion/index.tsx` :

```tsx
import { type ReactNode, useState } from 'react'
import { ChevronRight } from 'lucide-react'
import styles from './Accordion.module.css'

export interface AccordionSection {
  id: string
  title: string
  body: ReactNode
}

interface Props {
  sections: AccordionSection[]
  defaultOpenId?: string
  multiple?: boolean
}

export function Accordion({ sections, defaultOpenId, multiple = false }: Props) {
  const [openIds, setOpenIds] = useState<Set<string>>(
    new Set(defaultOpenId ? [defaultOpenId] : [sections[0]?.id].filter(Boolean) as string[])
  )

  const toggle = (id: string) => {
    setOpenIds(prev => {
      const next = new Set(multiple ? prev : [])
      if (prev.has(id)) next.delete(id)
      else next.add(id)
      return next
    })
  }

  return (
    <div className={styles.root}>
      {sections.map(s => {
        const open = openIds.has(s.id)
        return (
          <section key={s.id} className={styles.section}>
            <button
              type="button"
              className={[styles.header, open && styles.open].filter(Boolean).join(' ')}
              aria-expanded={open}
              aria-controls={`acc-body-${s.id}`}
              onClick={() => toggle(s.id)}
            >
              <ChevronRight size={14} className={styles.chevron} aria-hidden />
              <span>{s.title}</span>
            </button>
            {open && <div id={`acc-body-${s.id}`} className={styles.body}>{s.body}</div>}
          </section>
        )
      })}
    </div>
  )
}
```

**Choix** : une section ouverte à la fois (`multiple=false`). Section « Pièce » par défaut quand pièce active ; sinon « Paramètres ».

**CSS** : `.body { animation: expand 120ms ease-out; }`. Pas de transition de hauteur (auto height = piège). Apparition avec fade soft.

**`<MainPanel>`** orchestrateur :

```tsx
import { useAppSelector } from '@/hooks/redux'
import { selectActiveRoom } from '@/store/selectors'
import { Accordion } from '@/components/ui/Accordion'
import { PoseParamsForm } from '@/components/panels/PoseParamsForm'
import { PlankCatalog } from '@/components/panels/PlankCatalog'
import { SummaryTable } from '@/components/panels/SummaryDrawer/SummaryTable'
import { RoomMetaSection } from './RoomMetaSection'
import { RowsSection } from './RowsSection'

export function MainPanel() {
  const room = useAppSelector(selectActiveRoom)

  const sections = [
    ...(room ? [
      { id: 'room',  title: 'Pièce',     body: <RoomMetaSection roomId={room.id} /> },
      { id: 'rows',  title: 'Rangées',   body: <RowsSection roomId={room.id} /> },

  ]

  return <Accordion sections={sections} defaultOpenId={room ? 'room' : 'pose'} />
}
```

5. **Résumé matière** — `<SummaryTable />`. **Décision provisoire** : garder `SummaryDrawer` accessible + ajouter section accordéon. Trancher en PR review si redondance gênante.
