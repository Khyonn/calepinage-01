# Spec 16.4 — Mode unifié `main` + menu gauche accordéon + sélection canvas

## Contexte & objectif

Session S4 du jalon 16. Quatre changements fonctionnellement liés :

1. **Mode unifié `main`** : fusion `nav` + `edit`. Modes finaux : `main`, `plan`, `draw`. Le mode `main` couvre **deux URLs** :
   - `/:projectId/main` (sans pièce sélectionnée)
   - `/:projectId/main/:roomId` (avec pièce sélectionnée)
2. **Menu gauche en accordéon** : **le menu gauche** (côté contextuel) devient un accordéon de **3 sections** (dans cet ordre) :
   1. **Pièce active** — visible **uniquement si une pièce est sélectionnée**. Contient nom + yOffset + rangées (AddRowForm + RowList) + bouton « supprimer la pièce ». Auto-ouvre sur changement de pièce sélectionnée.
   2. **Paramètres de pose** — `<PoseParamsForm />`.
   3. **Catalogue** — `<PlankCatalog />`.
   Une seule section ouverte à la fois (`multiple=false`). État local (`useState`), pas de Redux.
3. **Sélection canvas** : suppression du dropdown / bouton « changer de pièce » dans la barre de mode. La sélection se fait par **clic sur la pièce dans le canvas** (mode `main`). Désélection par **clic sur le fond** du canvas. Garde-fou : un drag-release qui termine sur le fond ne doit **pas** désélectionner (cf B.4).
4. **Drawer droit (résumé)** : **pas de refonte structurelle**. Seule modification : retirer l'information « <nb> unités » qui est doublonnée dans le tableau (cf C). État d'ouverture du drawer reste local au composant porteur (cf 16.3).

### B. Menu gauche en accordéon + sélection canvas

#### B.1 Primitive `<Accordion>`

```tsx
import { type ReactNode, useState, useEffect } from 'react'
import { ChevronRight } from 'lucide-react'
import styles from './Accordion.module.css'

export interface AccordionSection {
  id: string
  title: string
  body: ReactNode
}

interface Props {
  sections: AccordionSection[]
  // mode contrôlé : si openId est passé, l'accordéon devient piloté par le parent (auto-ouverture sur changement de pièce active).
  openId?: string
  defaultOpenId?: string
  onOpenChange?: (id: string | null) => void
}

export function Accordion({ sections, openId, defaultOpenId, onOpenChange }: Props) {
  const [internalId, setInternalId] = useState<string | null>(defaultOpenId ?? sections[0]?.id ?? null)
  const isControlled = openId !== undefined
  const currentId = isControlled ? (openId ?? null) : internalId

  const toggle = (id: string) => {
    const next = currentId === id ? null : id
    if (!isControlled) setInternalId(next)
    onOpenChange?.(next)
  }

  return (
    <div className={styles.root}>
      {sections.map(s => {
        const open = currentId === s.id
        return (
          <section key={s.id} className={styles.section}>
            <button type="button"
              className={[styles.header, open && styles.open].filter(Boolean).join(' ')}
              aria-expanded={open}
              aria-controls={`acc-body-${s.id}`}
              onClick={() => toggle(s.id)}
            >
              <ChevronRight size={14} className={styles.chevron} aria-hidden />
              <span>{s.title}</span>
            </button>
            {open && <div id={`acc-body-${s.id}`} className={styles.body}>{s.body}</div>}
          </section>
        )
      })}
    </div>
  )
}
```

`multiple` retiré : une seule section ouverte à la fois (mode unique).

CSS : `.body { animation: expand 120ms ease-out; }`. Pas de transition de hauteur (auto height = piège). Apparition fade soft.

#### B.4 Sélection canvas (`useCanvasSelection`)

Hook co-localisé dans `Scene/`. Branché au mount. Gère :
- **Clic sur une pièce** (mode `main`) → `dispatch(setActiveRoom(roomId))` + middleware met à jour l'URL.
- **Clic sur le fond du canvas** → `dispatch(setActiveRoom(null))`.
- **Garde-fou drag-release** : un click qui résulte d'un drag (pointer down → bouge → pointer up sur fond) ne doit **pas** désélectionner. Détection : on n'écoute que `click` natifs où le `pointerdown` initial avait pour cible le **fond strictement** (background grid). Un drag commencé sur une `Row` / `Vertex` / `Segment` produit aussi un `click` final si le pointer se relâche sur le SVG, mais l'élément cible du `pointerdown` n'est pas le fond → on ignore.

```ts
import { useEffect, type RefObject } from 'react'
import { useAppDispatch, useAppSelector } from '@/hooks/redux'
import { uiActions } from '@/store/uiSlice'
import { selectInteractionMode, selectActiveRoomId } from '@/store/selectors'

interface Args {
  svgRef: RefObject<SVGSVGElement | null>
  // Sélecteur CSS / data-attribute identifiant le « fond » du canvas (ex. data-canvas-bg)
  backgroundSelector: string
}

export function useCanvasSelection({ svgRef, backgroundSelector }: Args): void {
  const dispatch = useAppDispatch()
  const mode = useAppSelector(selectInteractionMode)
  const activeRoomId = useAppSelector(selectActiveRoomId)

  useEffect(() => {
    if (mode !== 'main') return
    const svg = svgRef.current
    if (!svg) return

    let downOnBackground = false

    const onPointerDown = (e: PointerEvent) => {
      const t = e.target as Element | null
      downOnBackground = !!t?.matches(backgroundSelector)
    }

    const onClick = (e: MouseEvent) => {
      const t = e.target as Element | null
      if (!t) return
      // 1. Click sur pièce
      const roomNode = t.closest('[data-room-id]')
      if (roomNode) {
        const id = roomNode.getAttribute('data-room-id')
        if (id) dispatch(uiActions.setActiveRoom(id))
        return
      }
      // 2. Click sur fond — désélection seulement si le pointerdown a aussi atterri sur le fond
      if (downOnBackground && t.matches(backgroundSelector)) {
        if (activeRoomId !== null) dispatch(uiActions.setActiveRoom(null))
      }
    }

    svg.addEventListener('pointerdown', onPointerDown)
    svg.addEventListener('click', onClick)
    return () => {
      svg.removeEventListener('pointerdown', onPointerDown)
      svg.removeEventListener('click', onClick)
    }
  }, [dispatch, mode, activeRoomId, svgRef, backgroundSelector])
}
```

Conventions :
- Chaque `<Room>` SVG porte `data-room-id={room.id}` sur son élément cliquable principal.
- Le fond du canvas (rectangle pleine zone derrière les pièces, ou `<g class="background">` portant `data-canvas-bg>`) porte `data-canvas-bg` (ou classe dédiée). `backgroundSelector = '[data-canvas-bg]'`.
- `Row`, `Segment`, `VertexHandle` ne portent **pas** `data-canvas-bg` ni `data-room-id` (et ne sont pas leurs descendants directs si on veut éviter les bubbling) → `closest('[data-room-id]')` les ignore au niveau click pièce, et `t.matches(backgroundSelector)` est faux.
- Drag d'un sommet / segment qui finit hors pièce : `pointerdown` cible = vertex/segment, `downOnBackground = false`, click final ignoré côté désélection. ✓ garde-fou actif.

**Tests store-first** (cf section Tests) : couvrir « click pièce → activeRoomId change », « click background → activeRoomId = null », « drag-release après pointerdown sur row → activeRoomId inchangé ». Le hook est testé en interaction avec le store via simulation des callbacks (pas de DOM réel — on dispatche directement les actions résultantes et on vérifie la cohérence des sélecteurs).

- **Risque migration tests** : nombreux fichiers `.steps.ts` mentionnent `'nav'`/`'edit'`. Patch global puis relancer suite. Adapter à la main si setup ne match plus.
- **Risque sélection canvas cassée** : si `data-canvas-bg` ou `data-room-id` manquent dans le rendu SVG, le hook `useCanvasSelection` ne sélectionne / désélectionne plus. Vérifier le DOM SVG en DevTools avant push.
- **Risque garde-fou drag-release** : si un autre composant dispatche `setActiveRoom` au mouseup d'un drag, la désélection involontaire revient. Auditer tous les sites qui dispatchent `setActiveRoom` après ce refacto.
- **Risque routing rétrocompat** : URLs `'nav'`/`'edit'` ouvertes par bookmarks. Cas couvert par `resolveRoute` (D).
- **Risque `<details>` natif vs custom** : custom retenu pour contrôle animation et `openId` contrôlé (auto-ouverture).
