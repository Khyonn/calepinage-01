### 1. Détection URL dans description

Helper `src/core/utils/url.ts` :

```ts
const URL_REGEX = /https?:\/\/[^\s)<>]+/i

export function extractFirstUrl(text: string): string | null {
  const match = text.match(URL_REGEX)
  return match ? match[0] : null
}
```

Tests core (créer `src/core/__tests__/extract-url.feature` + steps) :
- URL simple → trouvée
- URL en milieu de phrase → trouvée
- pas d'URL → `null`
- multiple URLs → première
- URL avec parenthèses fermantes ou `<>` → exclues du match

### 3. Primitive Tooltip

Pas de portail React 19 nécessaire. Implémentation simple en pseudo-popover positionné absolument au-dessus du déclencheur, affiché via hover/focus.

```tsx
import { useState, useRef, type ReactNode } from 'react'
import styles from './Tooltip.module.css'

interface Props {
  content: string  // texte multi-lignes, retours ligne respectés
  children: ReactNode
}

export function Tooltip({ content, children }: Props) {
  const [open, setOpen] = useState(false)
  const ref = useRef<HTMLSpanElement>(null)

  return (
    <span
      ref={ref}
      className={styles.wrapper}
      onMouseEnter={() => setOpen(true)}
      onMouseLeave={() => setOpen(false)}
      onFocus={() => setOpen(true)}
      onBlur={() => setOpen(false)}
    >
      {children}
      {open && <span className={styles.bubble} role="tooltip">{content}</span>}
    </span>
  )
}
```

CSS : `whitespace: pre-wrap` sur `.bubble` pour respecter les retours ligne. Positionnement `position: absolute; bottom: 100%; left: 50%; transform: translateX(-50%)`. Largeur max `280px`.

### 5. Refonte CSV — structure cible

Le CSV de sortie comprend trois sections séparées par une ligne vide :

#### 5.1 Section « Résumé matière » (existe, simplifier)

```
# Résumé matière
Type de lame;Quantité;Coût unitaire (€);Coût total (€)
Chêne 120×12 cm;12;8.50;102.00
Total;;;102.00
```

`describePlank(plankType)` doit renvoyer `${name} ${length}×${width} cm` (drop `, N lames @ ...` qui était dans le label précédent).

#### 5.2 Section « Détail par pièce » (NOUVELLE)

Pour chaque pièce, un sous-bloc :

```
# Détail — <nom de la pièce>
Rangée;Type de lame;Segment;Première lame (cm);Source;Lames complètes;Dernière lame (cm);Destination
1;Chêne 120×12;1;30.0;rangée 0 de Salon;3;39.7;rangée 2 de Salon
1;Chêne 120×12;2;;;1;78.3;
2;...
```

Colonnes :

- **Rangée** : index 1-based.
- **Type de lame** : nom + dimensions.
- **Segment** : index 1-based dans la rangée. Une rangée peut avoir plusieurs segments (pièces concaves).
- **Première lame (cm)** : longueur de la première lame du segment **si elle est coupée**, sinon vide.
- **Source** : étiquette `rangée N de <pièce>` si la première lame provient d'une chute réutilisée, sinon vide.
- **Lames complètes** : nombre de lames pleines au milieu du segment, vide si zéro.
- **Dernière lame (cm)** : longueur de la dernière lame **si elle est coupée**, sinon vide.
- **Destination** : étiquette `rangée N de <pièce>` si la chute de la dernière lame est réutilisée ailleurs, sinon vide.

#### 5.3 Règles de colonnes vides

Pour chaque segment, déterminer les colonnes selon le contenu :

| Contenu segment | Première lame | Lames complètes | Dernière lame |
|---|---|---|---|
| Une seule lame, complète | vide | vide ou `1` (cohérent) | vide |
| Une seule lame, incomplète | longueur | vide | vide |
| Première complète, milieu plein, dernière complète | vide | nb total - 1 ? voir détail | vide |
| Première coupée, milieu plein, dernière coupée | longueur | nb complètes | longueur |
| Aucune complète (segment trop court pour une lame entière) | longueur | vide | vide ou longueur |

**Cas explicites** demandés par l'utilisateur :
- Si le segment commence par une lame complète → première lame = vide.
- Si le segment ne contient qu'une seule lame non complète → lames complètes + dernière lame = vides.
- Si le segment ne contient pas de lame complète → lames complètes = vide.
- Si la dernière lame est complète → dernière lame = vide.

Implémenter comme une fonction pure dans `core/exportCsv.ts` :
